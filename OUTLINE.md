# OUTLINE

## Chapter 1: デプロイの仕組みを理解する（3 Section）

**ゴール**: Webアプリが公開される仕組み（サーバー・DNS・HTTPS・ホスティング）を理解し、自分のポートフォリオに適したデプロイ手段を選べる

- **1-1 Webアプリが公開されるまでの全体像**
  - 種類: 概念
  - ゴール: ブラウザにURLを入力してからページが表示されるまでの流れ（DNS解決→サーバー→レスポンス）を説明できる
  - 要点: サーバーとは何か、DNSの役割、HTTP/HTTPSの違い、静的サイトと動的サイトの違い
  - 参考資料: [Vercel公式ドキュメント](https://vercel.com/docs)

- **1-2 ホスティングサービスの種類と選び方**
  - 種類: 概念
  - ゴール: 静的ホスティング・PaaS・IaaSの違いを理解し、用途に応じて選べる
  - 要点: 各カテゴリの特徴と代表サービス、React SPAが「静的サイト」として扱える理由、コスト比較の観点
  - 参考資料: [Vercel公式ドキュメント](https://vercel.com/docs)

- **1-3 Vercelを選ぶ理由**
  - 種類: 概念
  - ゴール: Vercelの特徴（GitHub連携・自動デプロイ・CDN・無料枠）を理解し、React SPAのデプロイ先としてVercelが適している理由を説明できる
  - 要点: Vercelのアーキテクチャ概要、Viteプロジェクトとの相性、無料プランの範囲と制限
  - 参考資料: [Vercel公式ドキュメント](https://vercel.com/docs)

## Chapter 2: Vercelでポートフォリオを公開する（3 Section）

**ゴール**: 自作のReact SPAをVercelにデプロイし、独自ドメインでHTTPS公開できる

- **2-1 アカウント作成とGitHub連携**
  - 種類: 混合
  - ゴール: Vercelアカウントを作成し、GitHubアカウントと連携できる
  - 要点: アカウント作成手順、GitHub連携の仕組み（Webhookによる自動デプロイ）、ダッシュボードの見方
  - 参考資料: [Vercel公式ドキュメント](https://vercel.com/docs)

- **2-2 React SPAをデプロイする**
  - 種類: ハンズオン
  - ゴール: GitHubリポジトリからReact (Vite) プロジェクトをVercelにデプロイし、公開URLでアクセスできる
  - 要点: プロジェクトのインポート、ビルド設定（Viteの場合）、デプロイログの読み方、プレビューデプロイ
  - 🏃 実践: 自分のポートフォリオリポジトリをVercelにデプロイする
  - 参考資料: [Vercel公式ドキュメント](https://vercel.com/docs)

- **2-3 独自ドメインとHTTPS**
  - 種類: 混合
  - ゴール: カスタムドメインを設定し、HTTPSで公開できる
  - 要点: ドメインの取得方法、DNS設定（Aレコード・CNAMEレコード）、VercelでのHTTPS自動設定（Let's Encrypt）、ドメイン設定のトラブルシューティング
  - 🏃 実践: カスタムドメインを設定してアクセスを確認する
  - 参考資料: [Vercel公式ドキュメント](https://vercel.com/docs)
