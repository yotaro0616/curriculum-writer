# プロンプトスタイルガイド

教材の概念図を Gemini Pro で生成する際のプロンプト構成ルール。

## 基本ルール

| 項目 | ルール |
|------|--------|
| 言語 | **ラベル・キーワードは日本語** で指定する。英語の技術用語（Event Loop, API 等）はそのまま使用可 |
| スタイル | フラットデザイン・インフォグラフィック風。影・グラデーション・写実表現は避ける |
| 背景 | 白背景（#FFFFFF） |
| 配色 | 青系（#3B82F6）とオレンジ系（#F97316）をアクセントカラーとする。対比図では左右で色を分ける |
| 線画 | アウトライン付きのクリーンな線画。手書き風にしない |
| 情報量 | 1枚に盛り込む概念は1つ。要素は5〜8個以内に抑える |
| 文字 | 大きく読みやすいラベルのみ。長文は入れない |

## プロンプト構成テンプレート

```
[画像の種類]を作成してください。

【内容】
- [何を描くか。比喩を使う場合はその対応関係を具体的に説明する]

【スタイル】
- フラットデザイン・インフォグラフィック風
- 白背景、青（#3B82F6）とオレンジ（#F97316）のアクセントカラー
- クリーンな線画、ラベルは日本語
- 16:9 横長レイアウト
```

**ポイント**: 【内容】セクションは具体的に書く。「イベントループの図」ではなく「中央に円形の回転ベルトコンベア、左側にマイクロタスクキューとタスクキュー、右側に Web APIs の作業場」のように、構図を明確に指定する。

## 画像の種類別ガイド

### メンタルモデル

抽象概念を日常の物（工場・建物・乗り物等）に置き換えて、「どういう感覚で捉えればいいか」を伝える。

- 中心に主概念を配置し、周囲に関連要素を配置する
- 比喩の対応関係をラベルで明示する（例: 「作業員 = シングルスレッド」）
- 技術用語と比喩を併記する

**例**: イベントループ → 回転ベルトコンベアと作業員、型消去 → 建設足場

### Before/After（比較図）

技術の「なぜ必要か」を、課題と解決の対比で視覚的に伝える。

- 左右に分割し、左に課題（暗い/乱雑）、右に解決（明るい/整理）を配置する
- 矢印で進化・転換を示す
- 左右で配色を分ける（左: グレー寄り、右: 青/オレンジのアクセント）

**例**: Fat Controller → Clean Architecture、SSR → SPA

### 鳥瞰図（アーキテクチャポスター）

複雑なシステムの全体像を「地図」として1枚に収める。Mermaid では要素数が多すぎて見づらくなる場合に使う。

- アイソメトリック（斜め上から見下ろす）視点を使う
- 全体の「形」を把握できる程度に簡略化する
- 各要素に日常の比喩ラベルを付ける（例: CloudFront = 料金所ゲート）

**例**: AWS 全体構成 → 街の鳥瞰図、通知システム → 都市マップ

### 比喩・アナロジー

技術概念と日常概念の対応を視覚的に明示する。🧠（先輩エンジニアの語り）で使われている比喩を絵にするのに適している。

- 技術側と日常側の両方をラベルで併記する
- 対応関係が一目でわかる構図にする

**例**: ジェネリクス → クッキー型、サイドカーパターン → バイクのサイドカー
