# 教材プロジェクトのセットアップ

CLAUDE.md の哲学（抽象）を OUTLINE.md の構造（具体）に落とし込む。

```
CLAUDE.md（哲学）          OUTLINE.md（構造）          writing.md（ルール）
 WHO  誰に教えるか    →    ペルソナに合った題材選定
 WHY  なぜ必要か      →    教材全体のストーリー
 WHAT 何ができるか    →    Part/Chapter/Section に MECE 分解
 HOW  どう教えるか    →    各 Section の種類・順序・依存関係
 MAP  リソースはどこか →                                   用語・トーン調整
```

**入力**: $ARGUMENTS

---

## Phase 1: 哲学の定義（CLAUDE.md）

AskQuestion で以下を順に質問し、回答を CLAUDE.md に書き込む。

### 1. WHO（ペルソナ）

- 読者像（職種・経験年数・立場）、動機、現在地
- 前提知識（カテゴリ別）
- 技術スタック（ツール・バージョン）

### 2. WHY（コンセプト）

- なぜ今このスキルが求められるか、読者にとっての価値、教材の哲学

### 3. WHAT（ゴール）

- 修了後に「何ができるか」を行動レベルで定義

### 4. HOW（カリキュラム大枠）

- Part 構成、各 Part のテーマと概要、ハンズオンの有無と方針

### 5. MAP（リソース）

- {{TOPIC}} の設定
- 公式ドキュメント URL
- 実践プロジェクトの有無と情報

---

## Phase 2: 構造化（OUTLINE.md）

WHAT と HOW を踏まえ、OUTLINE.md を作成する。

### 分解の手順

1. **WHAT → Part**: ゴールを能力/フェーズ単位で分解
2. **Part → Chapter**: 学習単元（トピック）で分解
3. **Chapter → Section**: 1回の学習で完結する粒度で分解
4. **Section の種類と公式ドキュメント URL を付与**

### MECE 設計原則

| 原則 | チェック |
|---|---|
| **階層的 MECE 分解** | 上位を漏れなく・重複なく分解しているか |
| **抽象度の一貫性** | 兄弟要素は同じ抽象度か |
| **各レベル 2〜4 個** | 5個以上ならグルーピング不足を疑う |
| **分解軸の単一性** | 兄弟が同じ軸上にあるか |

実践プロジェクトがある場合は、使用する Part/Chapter、Chapter 間の依存関係、各🏃 実践の概要も設計する。

ユーザーに OUTLINE.md を提示し、構成の承認を得る。

---

## Phase 3: ルール調整（writing.md）

1. **語りかけの人格**: `[人格名]` を具体化
2. **用語テーブル**: {{TOPIC}} 固有の用語を追加
3. **その他**: 必要に応じて絵文字・コード言語指定等を調整

---

## Phase 4: 確認

生成した3ファイル（CLAUDE.md・OUTLINE.md・writing.md）をユーザーに提示し、修正指示があれば反映する。
