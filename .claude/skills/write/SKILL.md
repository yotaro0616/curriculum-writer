# 教材の執筆ワークフロー

**入力**: $ARGUMENTS

## スコープの解釈

入力からスコープ（対象範囲）を判定する。

| 入力形式 | 対象 | 例 |
|---|---|---|
| `Part番号` or `番号` | Part 全体 | `1`, `Part 2` |
| `Part番号-Chapter番号` | Chapter | `2-1`, `Chapter 2-3` |
| `Part番号-Chapter番号-Section番号` | Section | `2-3-5`, `Section 1-1-2` |

ディレクトリパス: `curriculums/part-XX_タイトル/chapter-XX_タイトル/X-X-X_タイトル.md`
タイトルは `OUTLINE.md` から取得する。

スコープ以降のテキストは追加の執筆指示として反映する。

## 1. 準備

1. **哲学の確認**: `CLAUDE.md` を Read し、教材の哲学（ペルソナ・コンセプト・ゴール）を再確認する
2. **設計読み込み**: `OUTLINE.md` を Read し、対象スコープの設計を把握する（ゴール、種類、公式ドキュメント URL 等）
3. **情報取得**: 公式ドキュメント URL を WebFetch する。加えて WebSearch で関連情報を積極的にリサーチする
4. **既存コンテンツの確認**: `curriculums/` 配下に既存ファイルがある場合、サブエージェントで分析する（用語の一貫性、前後のスコープとの接続）
5. **実践プロジェクト固有の準備**（CLAUDE.md に実践プロジェクトが定義されている場合のみ）:
   - プロジェクトの仕様書を Read し、仕様（モデル、画面、タスク対応表等）を把握する
   - 対象 Section の🏃 実践に関わる実プロジェクトのコードをサブエージェントで確認する
   - **実践で言及するコード（ファイルパス・メソッド名・行の内容）は実在を検証してから書く**

## 2. 方針合わせ

ユーザーに判断を伝え、方向性を揃える。

- **体験設計**: この範囲で学習者にどういう体験を届けるか
- **構成判断**: 見出し構成を提案する。OUTLINE.md のスコープを変更する場合は理由を含める
- **不明点**: 判断に迷う点があれば質問する

ユーザーの反応を受けてから執筆に入る。

## 3. 執筆

CLAUDE.md の哲学を踏まえ、`.claude/rules/writing.md` のルール・テンプレートに沿って書く。

- **概念 Section**: 概念テンプレートに従う
- **ハンズオン Section**: ハンズオンテンプレートに従う。冒頭に概念 Section への逆リンク
- **混合 Section**: 混合テンプレートに従う。🏃 実践パートを含む

Part / Chapter スコープの場合は、含まれる全 Section を順に執筆する。

## 4. 検証・完了

1. **コード整合性検証**（実践プロジェクトを扱う場合）: 🏃 実践で言及したコード（ファイルパス、メソッド名、行の内容）がサブエージェントで実在するか確認する
2. **公式ドキュメント照合**: サブエージェントに公式ドキュメントを再取得させ、差異を報告させる
3. **レビュー**: `/review [スコープ]` を実行する
4. **ユーザー確認**: レビュー結果を報告する。軽微な修正は同時に修正してよい。設計変更はユーザー確認を取る
