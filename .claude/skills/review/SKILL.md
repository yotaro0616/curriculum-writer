# 教材のレビュー

問題点は自動修正せず、すべて報告のみとする。

**入力**: $ARGUMENTS

## スコープの解釈

| 入力形式 | 対象 | 例 |
|---|---|---|
| `Part番号` or `番号` | Part 全体 | `1`, `Part 2` |
| `Part番号-Chapter番号` | Chapter | `2-1`, `Chapter 2-3` |
| `Part番号-Chapter番号-Section番号` | Section | `2-3-5`, `Section 1-1-2` |

## 0. 並列化（Part 以上のスコープの場合）

Part ごとにサブエージェントを並列起動し、以下を渡す:
- OUTLINE.md の該当セクション、全 Section ファイルパス
- `.claude/rules/writing.md` の全ルール
- レビュー観点（セクション2の全項目）と出力フォーマット（セクション4）

各サブエージェントの結果を統合し、横断整合性チェック（セクション3）はメインで実行する。

## 1. 準備

1. `OUTLINE.md` を Read し、対象スコープのゴール・Section 構成を確認
2. 対象スコープと前後の Section ファイルを Read

## 2. 単体レビュー

以下の5観点で検証する。

### テンプレート準拠

`.claude/rules/writing.md` のテンプレートに準拠しているか:
- 🎯 / ✨ が揃っているか、`---` 区切りが適切か
- Section の種類に応じた追加要素（🔍 見極めチェック、🏃 実践、逆リンク等）があるか
- 絵文字が定義済みの範囲内か

### 設計との整合

- Section 構成・ゴール・種類が OUTLINE.md と一致しているか

### 正確性

- {{TOPIC}} の用語が公式ドキュメントの表記に従っているか
- コードブロックの言語指定が正しいか

> 📝 公式ドキュメントとの詳細な照合は `/check-updates` の責務。

### スタイル準拠

- 敬体で統一されているか、読者への呼びかけが「あなた」か

### 実践フォロー可能性

🏃 実践・ハンズオンを含む Section が対象。読者が記載内容だけで最後まで進められるかを検証する:

- 前提条件（環境・ツール・ログイン情報等）が 📌 で明記されているか
- 手順の整合性（コマンドと説明の矛盾、Step の飛躍がないか）
- 前の Chapter/Section の成果物への依存が明示されているか
- 外部リソースへの依存がある場合、代替パスが示されているか
- 実践プロジェクトのコード参照（ファイルパス・メソッド名）が実在するか

### リンク・参照の検証

- Section 間リンクの参照先が実在するか
- 本文中の Section 番号が実際の番号・タイトルと一致しているか

## 3. 横断整合性チェック

> 対象スコープ外に既存コンテンツが存在する場合のみ実行。

- 用語の一貫性、文体の統一、Section 間の接続の自然さ

## 4. 出力

```
**[スコープ] レビュー結果**: 指摘 X 件（🔴 X / 🟡 X / 🔵 X）

## 修正アクションリスト

| # | 重要度 | ファイル | 箇所 | 修正内容 |
|---|---|---|---|---|
```

重要度: 🔴 技術的誤り / 🟡 設計不整合 / 🔵 軽微なスタイル問題
