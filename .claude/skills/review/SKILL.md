# 教材のレビュー

問題点は自動修正せず、すべて報告のみとする。

**入力**: $ARGUMENTS

## スコープの解釈

CLAUDE.md の HOW セクションで定義された階層構造に基づいてスコープを判定する（`/write` と同じルール）。

## 0. 並列化（上位スコープの場合）

上位スコープの場合、グループごとにサブエージェントを並列起動する。各サブエージェントには OUTLINE.md の該当部分、対象ファイルパス、レビュー観点と出力フォーマットを渡す。

各サブエージェントの結果を統合し、横断整合性チェック（セクション3）はメインで実行する。

## 1. 準備

1. `OUTLINE.md` を Read し、対象スコープのゴール・Section 構成・参考資料を確認
2. 対象スコープと前後の Section ファイルを Read
3. OUTLINE.md に記載された参考資料を取得する。**ローカルファイルパスは Read ツールで読み込む**。URL がある場合は WebFetch で取得する。教材の記述内容と参考資料の実態を照合するために使用する

## 1.5 機械的チェック（Grep/Bash ベース）

サブエージェント起動前に、プログラムで検出可能なルール違反を Bash で一括検証する。結果をサブエージェントに渡し、重複チェックを避ける。

対象ファイルに対し、Grep ツール（ripgrep ベース）で以下を検証する。Bash の `grep` コマンドは macOS 互換性の問題があるため使用しない。

| ルール | Grep パターン / 検証方法 |
|---|---|
| `**` 直後スペース欠落 | `\*\*[^\s*\n]`（閉じ `**` の直後にスペースも改行もない箇所） |
| 文全体の太字 | `\*\*[^*]{30,}\*\*` や絵文字行内の長い太字範囲を検出 |
| ダッシュ記号の使用 | `[—–]` |
| 言語指定なしコードブロック | `` ^```$ ``（` ``` ` のみの行） |
| writing.md 未定義の絵文字 | 定義済み絵文字（🎯✨💡⚠️📝🔑🏃🧠📌📖✅）以外の絵文字を検出 |
| 絶対パスの混入 | `/Users/` や `/home/` を検出 |
| 🎯 / ✨ の欠落 | 各 Section ファイルに `## 🎯` と `## ✨` が1つ以上あるか確認 |
| 前提知識の欠落 | OUTLINE の `前提:` フィールドを持つ Section に `📝 **前提知識**` があるか確認（🟡） |
| 接続文のキーワード | 各 Section 末尾の接続文が OUTLINE の次 Section ゴールキーワードを含むか確認（🟡） |

機械的チェックの結果は 🔵（軽微なスタイル問題）として出力に含める。ただし上記で（🟡）と記載した項目は設計不整合として扱う。

## 2. 単体レビュー

以下の4観点で検証する。各観点の具体的なチェック項目は `.claude/rules/writing.md` を参照すること（ここでは重複して列挙しない）。

### ルール準拠

`.claude/rules/writing.md` に定義された文体・テンプレート・コンテンツ規則に準拠しているか

### 設計との整合

`OUTLINE.md` の設計（Section 構成・ゴール・種類）と実際の内容が一致しているか

### 正確性

{{TOPIC}} の用語・内容が参考資料の表記に従っているか

> 📝 参考資料との詳細な照合は `/check-updates` の責務。

### 実践フォロー可能性

🏃 実践・ハンズオンを含む Section が対象。以下を検証する:

- **状態の追跡**: 各ステップ実行後の Git 状態・ファイル状態が、次ステップの前提と矛盾しないか
- **コマンドの正確性**: そのステップの時点の状態でコマンドが期待どおりに動作するか
- **前提条件の明示**: 📌 チェックリストで実行前の必要状態が網羅されているか
- **副作用のクリーンアップ**: 実践で変更したリソースを元に戻す手順が含まれているか

## 3. 横断整合性チェック

> 対象スコープ外に既存コンテンツが存在する場合のみ実行。

- 用語の一貫性、文体の統一、Section 間の接続の自然さ
- リンク・参照先の実在確認、セクション番号の整合

## 4. 出力

```
**[スコープ] レビュー結果**: 指摘 X 件（🔴 X / 🟡 X / 🔵 X）

## 修正アクションリスト

| # | 重要度 | ファイル | 箇所 | 修正内容 |
|---|---|---|---|---|
```

重要度: 🔴 技術的誤り / 🟡 設計不整合 / 🔵 軽微なスタイル問題
