# 最新情報の差分チェック

{{TOPIC}} の最新ドキュメント・Changelog と教材を照合し、更新が必要な箇所を報告する。自動修正はしない。

**入力**: $ARGUMENTS

**解釈ルール**: スコープ指定時はその Chapter のみ、未指定時は全 Chapter を対象にチェック。

**前提**: `OUTLINE.md` の参考文献 URL を照合基準とする。Chapter ファイルが存在する場合のみ鮮度チェックを行う。

## ステップ1: 情報源の取得

以下を WebFetch で**並列に**取得:
1. 公式ドキュメントのページ一覧（llms.txt 等）
2. Changelog（直近3バージョン重点）
3. OUTLINE.md の参考文献 URL

WebFetch 失敗時は失敗 URL と理由を記載し、取得できた情報源のみで続行。

## ステップ2: ドキュメント構成の差分チェック

ドキュメント一覧と OUTLINE.md を照合:
- **新規ページ**: 教材にない → 既存 Section への追加 or 新規 Section 候補
- **削除・統合**: 参考文献にあるがドキュメント一覧にない

## ステップ3: Changelog との差分チェック

- API・コマンド・設定の変更（廃止、新追加、構文変更）
- 新機能の追加（教材スコープ内）
- UI・操作手順の変更
- 破壊的変更（最優先で報告）

## ステップ4: 参考文献 URL の有効性チェック

ドキュメント一覧に含まれない URL のみ WebFetch で確認。リダイレクトがあれば記録。

## ステップ5: 結果の報告

チェック実施日、Changelog 最新バージョン、破壊的変更、未反映の新機能・変更、ドキュメント構成の変更、参考文献 URL の状態、推奨アクションを報告。

各項目に優先度を付与: 高（既存解説が誤りになる）/ 中（主要新機能）/ 低（マイナー改善）

---

## メンテナンス運用

- `/check-updates` を**月1回**実行。メジャーアップデート時は即座に実行
- 破壊的変更 → 即座に `/write` で修正
- 構成変更が必要な場合は `/setup` を使用
