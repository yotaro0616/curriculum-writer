# 参考資料との差分チェック

教材で扱う技術 の参考資料と教材の内容を照合し、更新が必要な箇所を報告する。自動修正はしない。

**入力**: $ARGUMENTS

**解釈ルール**: スコープ指定時はその範囲のみ、未指定時は全体を対象にチェック。スコープの解釈は `/write` と同じルール。

**前提**: `OUTLINE.md` の参考資料を照合基準とする。Section ファイルが存在する場合のみ鮮度チェックを行う。

## ステップ1: 情報源の取得

OUTLINE.md に記載された参考資料を確認し、取得可能なものを**並列に**取得する:

- **ローカルファイルパスは Read ツールで読み込む**（例: `/Users/yotaro/lms/...` のパスは実際のコードを Read して最新状態を確認する）
- URL がある場合: WebFetch で取得（公式ドキュメント、Changelog 等）
- URL がない場合（書籍等）: WebSearch で最新版・正誤表・改訂情報を検索

WebFetch 失敗時は失敗 URL と理由を記載し、取得できた情報源のみで続行。

## ステップ2: 参考資料との差分チェック

取得した情報と OUTLINE.md・教材本文を照合:
- **内容の変更**: 教材の記述が古くなっている箇所
- **新規追加**: 参考資料にあるが教材にない内容 → 既存 Section への追加 or 新規 Section 候補
- **削除・統合**: 参考資料から消えた内容
- **破壊的変更**: 既存の解説が誤りになるもの（最優先で報告）

## ステップ3: 参考資料 URL の有効性チェック

URL がある参考資料のリンク切れ・リダイレクトを確認。

## ステップ4: 結果の報告

チェック実施日、情報源の取得状況、破壊的変更、未反映の変更、参考資料の状態、推奨アクションを報告。

各項目に重要度を付与: 🔴（既存解説が誤りになる）/ 🟡（主要な変更）/ 🔵（マイナー改善）

---

## メンテナンス運用

- 参考資料がオンラインの場合: `/check-updates` を**月1回**実行。メジャーアップデート時は即座に実行
- 破壊的変更 → 即座に `/write` で修正
- 構成変更が必要な場合は `/setup` を使用
