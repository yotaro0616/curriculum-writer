# 執筆ルール

## 1. 文体

### トーン

- **本文**: 落ち着いた敬体で丁寧に解説する。過度に個人的な語りかけ（「率直に言うと」等）は避ける
- **🧠 [人格名]はこう考える**: 実体験ベースの語りをここに集中させる <!-- /setup で人格名を設定 -->

### 用語・表記

- 敬体（です・ます調）で統一。読者への呼びかけは「あなた」
- ダッシュ記号（`——`、`—`、`–`）は使わない
- {{TOPIC}} の用語は参考資料の表記に従う。範囲外の内容は WebSearch / WebFetch で裏付ける
- 時間経過で変わる情報は最新情報を取得し「○○時点」と日付を明記
- 外部ソースを参照する場合は出典を明記する
- `**太字**` は語句・キーフレーズに限定する。文全体を太字にしない
- Markdown の `**text**` は閉じ `**` 直後にスペースなしで日本語が続くと適用されない。`**○○** の場合` のようにスペースを入れる

| 表記 | 備考 |
|---|---|
| セクション | 教材本文ではカタカナ表記。内部ドキュメントでは英語 |

### 絵文字

| 絵文字 | 用途 |
|---|---|
| 🎯 | 学習目標（Section 冒頭） |
| ✨ | まとめ（Section 末尾） |
| 💡 | TIP・補足 |
| ⚠️ | 注意事項・よくあるエラー |
| 📝 | ノート・定義・補足説明 |
| 🔑 | 重要概念・キーポイント |
| 🏃 | ハンズオン手順・実践 |
| 🧠 | 経験者の視点・思考プロセス |
| 📌 | 操作前の確認事項 |
| ✅ | 完成チェックリスト（ハンズオン・混合） |
| 📖 | この Chapter の進め方（2層以上の場合） |

## 2. 構造

### Section の種類

| 種類 | 内容 |
|---|---|
| **概念** | 意義・仕組み・使い方を解説 |
| **ハンズオン** | 概念で学んだ機能を実践。冒頭に概念 Section への逆リンク |
| **混合** | 概念を学びながらすぐに手を動かす |

### 見出しの設計

**Why → What → How**（抽象→具体）の流れで組む。

### テンプレート

全 Section 共通の骨格に、種類ごとの差分を組み合わせる。

```markdown
# [セクション番号] セクションタイトル

<!-- ハンズオンの場合: 📝 行を挿入 -->
<!-- 概念（グループ先頭）の場合: グループ全体像を挿入 -->

## 🎯 このセクションで学ぶこと

- 学習目標1
- 学習目標2

[1文で Section の流れを示す]

---

## 導入: [課題・痛みから入る見出し]

### 🧠 [人格名]はこう考える

> [実務の経験に基づいて自由に語る]

---

<!-- 本文: 種類ごとに異なる（「種類ごとの差分」を参照） -->

---

## ✨ まとめ

- ポイント1
- ポイント2

---

次のセクションでは、[次の内容への接続]。
```

**次セクションへの接続**:
- プレーンテキストで記述する（Markdown リンクやカギ括弧は使わない）
- **最終セクションの場合**: 「次のセクション」リンクの代わりに、教材全体の締め括り文を記述する（学んだことの振り返り + 次のステップへの示唆）

### 種類ごとの差分

**概念**:

- 本文は `## 見出し` で構成する
- 2層以上の場合、親グループの最初の Section のみ、タイトル直後にグループ全体像（概要・セクション一覧・📖 進め方）を含める

**ハンズオン**:

- タイトル直後に `📝 **このハンズオンで使う機能**: [機能名]（[セクション名] で学習）` を挿入
- 導入の後に `## 📌 [作業名]前の確認`（チェックリスト形式）を挿入
- 本文は `## 🏃 実践: [タイトル]` の中に `### 🏃 Step N: [タイトル]` の連番で構成する
- 本文の後、まとめの前に `## ✅ 完成チェックリスト` を挿入
- 長いコードは `<details>` で折りたたむ

**混合**:

- 本文の前半は `## 見出し`（概念解説）で構成する
- 本文の後半は `## 🏃 実践: [タイトル]` の中に `### 🏃 Step N: [タイトル]` の連番で構成する
- 本文の後、まとめの前に `## ✅ 完成チェックリスト` を挿入

**エラーフォーマットの使い分け**:
- エラーメッセージ（コードブロック）を伴う場合 → blockquote 形式（下記テンプレート）
- エラーメッセージなしの一般的な注意 → `⚠️ **注意**: ...` でインラインに記述

> ⚠️ **よくあるエラー**: [概要]
>
> ```
> [エラーメッセージ例]
> ```
>
> **原因**: [なぜ発生するか]
>
> **対処法**: [解決手順]

## 3. コンテンツ

### コードの見せ方

- コードを含む場合、コードブロック上部にファイルパスをコメントで明示
- diff 表示は使わない（テキストで説明し変更後のコードを示す）

### 解説の深さ

- 前提知識は基本説明を省略。ただし {{TOPIC}} と組み合わせたときの注意点は丁寧に
- {{TOPIC}} 固有の概念は初出時に丁寧に解説し、2回目以降は初出 Section への参照
- 同じ概念を別の箇所で繰り返さない
- 導入は「これがないと何が困るか」（課題）から入る
- 処理の流れやサイクルは図で視覚化する <!-- /setup で形式を設定: ASCII or Mermaid -->
- つまずきポイントを先回りする（⚠️ で説明）
- 先の Section で扱う内容は「今は概要だけ把握すれば十分です」と明記する
- 初回リリースでは画像を含めず `<!-- TODO: 画像追加 - ○○の画面 -->` でプレースホルダーを残す
