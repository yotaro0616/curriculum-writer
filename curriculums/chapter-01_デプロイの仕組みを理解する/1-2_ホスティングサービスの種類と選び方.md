## ホスティングサービスの3つのカ���ゴリ

ホスティングサービスは、自由度と管理の手間によって大きく3つに分類できます。

### 静的ホスティング

ビルド済みの HTML・CSS・JavaScript ファイルをそのまま配信するサービスです。サーバーサイドのプログラム実行はできません。

| 特徴 | 内容 |
|---|---|
| できること | 静的ファイルの配信、CDN（Content Delivery Network）による高速配信 |
| できないこと | サーバーサイドのプログラム実行、データベース接続 |
| 代表サービス | Vercel、Netlify、GitHub Pages、Cloudflare Pages |
| コスト | 無料プランが充実（個人利用なら十分） |
| 学習コスト | 低い。Git連携で自動デプロイされる |

📝 **CDN とは**: CDN（Content Delivery Network）は、世界中に分散配置されたサーバー群です。ユーザーに地理的に近いサーバーからファイルを配信するため、表示速度が向上します。静的ホスティングサービスの多くは CDN を標準で備えています。

### PaaS

PaaS（Platform as a Service）は、アプリケーションの実行環境を提供するサービスです。サーバーの管理はサービス側が行い、開発者はコードをデプロイするだけです。

| 特徴 | 内容 |
|---|---|
| できること | サーバーサイドプログラムの実行、データベース接続 |
| できないこと | OS やミドルウェアの細かい設定 |
| 代表サービス | Render、Railway、Fly.io、Heroku |
| コスト | 無料枠あり。本格利用は月数ドル〜 |
| 学習コスト | 中程度。サービス固有の設定を覚える必要がある |

💡 **補足**: Laravel などのサーバーサイドアプリケーションをデプロイする場合は PaaS が適しています。ただし、React SPA のように静的ファイルだけを配信するなら、PaaS はオーバースペックです。

### IaaS

IaaS（Infrastructure as a Service）は、仮想サーバー（コンピュータ）そのものを提供するサービスです。OS のインストールから自分で行うため、自由度が最も高い反面、管理の手間も最大です。

| 特徴 | 内容 |
|---|---|
| できること | ほぼすべて。OS・ミドルウェア・ネットワークを自由に構成 |
| できないこと | 物理サーバーの管理（それはデータセンターの仕事） |
| 代��サービス | AWS EC2、Google Compute Engine、Azure VM、さくらVPS |
| コスト | 無料枠あり（AWS EC2 は12ヶ月無料）。本格利用は月数千円〜 |
| 学習コスト | 高い。Linux・ネットワーク・セキュリティの知識が必要 |

## React SPA には静的ホスティングが最適な理由

セクション 1-1 で学んだ通り、React SPA はビルドすると `dist/` フォルダに静的ファイルが生成されます。サーバーサイドの処理は必要ありません。つまり:

1. **必要なのはファイル配信だけ**: 静的ホスティングで十分
2. **コストがゼロ**: 無料プランでポートフォリオ公開に十分な容量・帯域
3. **管理の手間がない**: サーバーのセキュリティパッチやOSアップデートを気にしなくていい
4. **CDN が標準装備**: 世界中から高速にアクセスできる

⚠️ **注意**: 将来 Laravel API と連携するフルスタック構成にする場合、バックエンド側は PaaS や IaaS が必要になります。ただし、フロントエンドの静的ファイルは引き続き静的ホスティングで配信するのが一般的です。フロントとバックエンドのデプロイ先を分けるのは、実務でもよくあるパターンです。

## 静的ホスティングサービスの選び方

静的ホスティングと決めても、Vercel・Netlify・GitHub Pages・Cloudflare Pages など複数の選択肢があります。以下の観点で比較しましょう。

| 観点 | Vercel | Netlify | GitHub Pages | Cloudflare Pages |
|---|---|---|---|---|
| **React / Vite 対応** | ⭕ 自動検出 | ⭕ 自動検出 | 🔺 手動設定 | ⭕ 自動検出 |
| **Git 連携自動デプロイ** | ⭕ | ⭕ | ⭕ | ⭕ |
| **プレビューデプロイ** | ⭕ | ⭕ | ❌ | ⭕ |
| **カスタムドメイン** | ⭕ 無料 | ⭕ 無料 | ⭕ 無料 | ⭕ 無料 |
| **HTTPS** | ⭕ 自動 | ⭕ 自動 | ⭕ 自動 | ⭕ 自動 |
| **無料枠の帯域** | 100 GB/月 | 100 GB/月 | 制限なし | 無制限 |

どのサービスも基本的な機能は揃っています。この教材では **Vercel** を使います。その理由は次のセクションで詳しく説明します。

## ✨ まとめ

- ホスティングサービスは **静的ホスティング**・**PaaS**・**IaaS** の3カテゴリに分類できる
- React SPA は静的ファイルだけで動作するため、**静的ホスティングが最適**
- 静的ホスティングは無料・管理不要・CDN付きで、ポートフォリオ公開に最適
- 主要な静的ホスティングサービス（Vercel、Netlify、GitHub Pages、Cloudflare Pages）は基本機能に大きな差はない
