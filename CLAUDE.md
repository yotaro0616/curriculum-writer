# はじめてのデプロイ — Vercelでポートフォリオを公開しよう

> COACHTECH受講生が、Vercelを使ってReact SPAのポートフォリオをインターネットに公開できるようになる教材

## ペルソナ（WHO）

COACHTECHのLaravel教材を完了した受講生。React (Vite) でSPAを開発できるが、インフラの知識がなく、作ったアプリを公開した経験がない。ポートフォリオとして自作アプリをデプロイし、就職活動に活用したい。

### 前提知識

- **フロントエンド**: React, Vite, SPA開発
- **バックエンド**: Laravel（実務入門レベル。この教材では不使用）
- **ツール**: Git, GitHub の基本操作

### 技術スタック

- **Vercel**: 無料プランを使用
- **Node.js**: Viteビルドに必要（ローカル開発環境に導入済みを想定）

## コンセプト（WHY）

「アプリは作れるのに、公開できない」。これはインフラ知識のないWeb開発学習者が必ずぶつかる壁である。既存のデプロイ教材はインフラ経験者向けか完全初心者向けに偏っており、「SPAは作れるがインフラは未経験」という層にフィットするものが少ない。この教材は、デプロイの裏側で何が起きているかを理解しながら、実際にポートフォリオを公開するまでを一気通貫で学ぶ。

## ゴール（WHAT）

1. **デプロイの基礎概念を理解できる**: DNS・HTTPS・ホスティングサービスの種類など、Webアプリが公開される仕組みを説明できる
2. **React SPAをVercelで公開できる**: GitHubリポジトリと連携し、独自ドメインでポートフォリオを公開できる

## カリキュラム（HOW）

**階層構造**: 2層（Chapter > Section）

| Chapter | 内容 | Section数 |
|---|---|---|
| 1. デプロイの仕組みを理解する | DNS・HTTPS・ホスティングの基礎概念 | 3 |
| 2. Vercelでポートフォリオを公開する | アカウント作成からデプロイ・ドメイン設定まで | 3 |

各層の設計詳細は `OUTLINE.md` を参照。

CLAUDE.md は教材の哲学（WHO / WHY / WHAT / HOW）を定義し、`OUTLINE.md` はその哲学を具体的な設計に落とし込む。執筆上の判断（題材の選択・構成のアレンジ・外部調査）は `OUTLINE.md` の設計に従いつつ、臨機応変に行うこと。

## プロジェクトマップ（MAP）

執筆ルールは `.claude/rules/writing.md` を参照。

### Skills

| Skill | 用途 |
|---|---|
| `/setup` | 初期設定（CLAUDE.md・OUTLINE.md・writing.md の作成） |
| `/write` | 執筆（任意の階層単位） |
| `/review` | レビュー（品質・整合性チェック） |
| `/check-updates` | 公式ドキュメントとの鮮度チェック |
